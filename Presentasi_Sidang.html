Latar Belakang Masalah

  • Perhitungan finansial kontrak (BOQ) dan pembuatan dokumen BAST masih manual menggunakan Excel.
  • Sering terjadi kesalahan perhitungan rumus (Human Error) yang merugikan perusahaan.
  • Pengajuan Pekerjaan Tambah/Kurang (CCO) dari Klien tidak terdokumentasi secara terpusat.
  • Sulitnya melacak progres verifikasi pekerjaan dari Site Manager di lapangan.

Tujuan & Batasan Sistem

Tujuan Utama: Mendigitalisasi alur persetujuan (approval) proyek secara end-to-end (dari Draft BOQ hingga BAST) untuk mencegah kecurangan harga.


Batasan Masalah:

  • Sistem berfokus pada administrasi dokumen, bukan pada manajemen inventaris gudang fisik.
  • Melibatkan 4 Hak Akses utama: Admin, Klien, Site Manager, dan Direktur.

Metodologi Pengembangan

Pengembangan sistem menggunakan metode SDLC (Software Development Life Cycle) dengan pendekatan Agile/Waterfall.

  • Analisis: Mengumpulkan kebutuhan dari PT dan Klien.
  • Desain: Membuat Flowchart, Use Case, dan Wireframe UI.
  • Implementasi: Proses coding menggunakan Framework terbaru.
  • Testing: Melakukan uji coba menyeluruh (Black Box & White Box).

Arsitektur Teknologi (Tech Stack)

  • Backend: PHP 8 dengan Framework Laravel 11.
  • Frontend: HTML5, CSS3, Bootstrap 5 dengan konsep Glassmorphism UI.
  • Database: MySQL Relational (Mendukung SoftDeletes).
  • Inovasi: Menggunakan Fuzzy Match Algorithm AI untuk membaca & memvalidasi file Excel secara otomatis.
  • Keamanan: Enkripsi Bcrypt, Proteksi CSRF, & Middleware Multi-Role.

Pengujian Sistem

Aplikasi telah lolos 2 tahap pengujian standar industri:

  • Black Box Testing: Pengujian fungsionalitas UI (Upload File, Login, CRUD Data) dipastikan berjalan 100% tanpa error.
  • White Box Testing: Menggunakan Automated PHPUnit Testing. Telah menjalankan 7 skenario logika internal (termasuk Null Injection & Bypass Akses) dengan hasil PASS (Berhasil).

Kesimpulan

  • Sistem Administrasi BOQ Enterprise ini terbukti mampu mempercepat alur persetujuan dokumen proyek secara drastis.
  • Meminimalisir kesalahan perhitungan (Human Error) karena sistem memvalidasi data Excel langsung dengan Master Data Database.
  • Meningkatkan transparansi antara Klien dan Perusahaan melalui fitur Customer Portal & Tiket Pemeliharaan.
